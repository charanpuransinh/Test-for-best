# Project: T-Pro Framework
### Version: 2.0.4 (Stable)

## Overview
यह एक सुरक्षित और मॉड्युलर इंटरफेस सिस्टम है जिसे डेटा विज़ुअलाइज़ेशन और कंट्रोल मैनेजमेंट के लिए डिज़ाइन किया गया है। बाहर से देखने पर यह एक साधारण HTML-आधारित डैशबोर्ड लगता है, लेकिन इसके पीछे "Trishul Pro" का कोर इंजन काम करता है।

---

## Core Pipeline (System Logic)

सिस्टम एक 'Hub and Spoke' मॉडल पर काम करता है जहाँ `index.html` मेन एंट्री गेट है और `raksha-core.js` इसका दिमाग है।

### 1. The Frontend Layers (UI Components)
पूरी एप्लिकेशन को स्टेप-बाय-स्टेप मॉड्यूल्स में बांटा गया है:
* **1_home_page.html**: लैंडिंग ज़ोन और ओवरव्यू।
* **2_power_box.html**: कैलकुलेशन इंजन और स्ट्रैटेजी पैरामीटर्स (ATR/Fibonacci लॉजिक यहीं लोड होता है)।
* **3_scanner.html**: रियल-टाइम डेटा फिल्टरिंग और सिग्नल डिटेक्शन।
* **4_chart_page.html**: विज़ुअल एनालिसिस और टेक्निकल इंडिकेटर्स का रेंडरिंग।
* **5_watchlist.html**: एक्टिव एसेट्स की मॉनिटरिंग।
* **6_control_panel.html**: एडमिन सेटिंग्स और सिस्टम कॉन्फ़िगरेशन।

### 2. The Logic Backbone
* **raksha-core.js**: यह फाइल पूरी सुरक्षा और डेटा फ्लो को संभालती है। यह सुनिश्चित करती है कि यूज़र ऑथेंटिकेशन और डेटा प्रोसेसिंग के बीच कोई लीकेज न हो।
* **server.py**: बैकएंड गेटवे। यह API एंडपॉइंट्स को मैनेज करता है और रियल-टाइम डेटा को `raksha-core` तक पहुँचाता है।

### 3. Execution Workflow (Pipe-line)
1.  **Initialize**: `server.py` डेटा स्ट्रीम शुरू करता है।
2.  **Auth/Validation**: `raksha-core.js` सिस्टम की सिक्योरिटी चेक करता है।
3.  **Process**: `Power Box` और `Scanner` मॉड्यूल डेटा को फिल्टर करते हैं।
4.  **Display**: प्रोसेस किया हुआ डेटा `index.html` के जरिए संबंधित पेजों (Chart/Watchlist) पर रेंडर होता है।
